# 2026-03-16

## Deploy issue: Prisma schema push vs migrate

Deploy failed because `prisma db push` refused to add the `username` column with `@unique` constraint without `--accept-data-loss`. Had to manually add the column and index to the DB, then redeploy.

**Decision**: Switch to `prisma migrate deploy` for production going forward. Need to initialize the migrations directory and update the Dockerfile startup command.

## What was built (2026-03-15 evening session)

### Sync filter refinements
- Added `responseStatus` and `showAs` normalization to both Google and Outlook providers — Google stores attendee self-response + transparency, Outlook stores `responseStatus.response` + `showAs`
- Normalized to common values: `accepted`/`declined`/`tentative`/`needsAction` for response, `free`/`tentative`/`busy` for show-as
- Two new target sync toggles: `skipDeclined` (default on), `skipFree` (default off)
- Also added `skipWorkLocation` (default on) and `skipSingleDayAllDay` (default off) — single-day detection uses <= 24h duration check (timezone offset doesn't affect stored duration)
- Re-enabled `cloneToTarget` which had been disabled due to duplicate creation issues

### Configurable sync period
- Users can choose 30/60/90 days in advance for target calendar sync
- `syncDaysInAdvance` column on `CalendarEntry`, filtered at clone time not fetch time (keeps source data complete for calendar view)

### Booking page (Calendly-style)
- Full public booking flow: `/book/<username>/<slug>` — date picker → time slots → form → confirmation
- Slot computation: working hours (AvailabilityRule, Mon-Fri 09:00-17:00 default) minus busy calendar events minus existing bookings, divided into duration-sized slots
- Double-booking prevention: re-checks availability at POST time
- Booking creates event on host's target calendar (or first writable) with `[Booking] Guest — Event Type` title
- New models: `EventType` (with auto-slug), `AvailabilityRule`, `Booking`
- Added `username` field to User for public URLs
- Settings page at `/settings/booking` for event types CRUD, availability editor, username config
- `/bookings` page for management with cancel (deletes provider event)

### Calendar UI fix
- Timegrid event text overlap fixed with `overflow: hidden` + `text-overflow: ellipsis` on `.fc-timegrid-event`

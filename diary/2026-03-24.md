# 2026-03-24

## Session: Sync dedup fix + calendar view bug + event type icons + branding feature

### Key decisions and discoveries

**Sync oversync root cause:** The `cloneToTarget()` dedup relied solely on `TargetEventMapping` DB records. Same real-world event across multiple source calendars (different `Event` rows in DB) would both get synced. Fixed with fingerprint-based dedup (`title|startTime|endTime`) that checks already-mapped events before creating. Also added cleanup when mapping insert fails after event creation (previously left orphaned target events).

**Calendar view state mismatch:** `currentView` state initialized to `"dayGridMonth"` but FullCalendar's `initialView` was `"timeGridWeek"`. Month button appeared selected while week view was showing.

**Event type action buttons redesign:** Replaced text links (Preview/Edit/Delete) with icon buttons + tooltips. Preview now opens the actual booking page (`/book/{username}/{slug}`) in a new tab instead of an internal preview route. Added separate availability calendar icon.

**Branding feature — URL-only images:** Decided against file upload infrastructure for v1. Users paste URLs for profile photo and background image. `avatarUrl` field already existed on User (from OAuth) but was never displayed on booking pages. Added `brandColor`, `accentColor`, `backgroundUrl` fields to User model.

**Branding theming approach:** Used inline `style` props with brand colors rather than CSS custom properties or dynamic Tailwind classes. Simpler and avoids Tailwind's build-time class purging. Brand color applied to: selected dates, "Book appointment" button, redirect button, slot borders (accent).

**Background image dimming:** Used `linear-gradient(rgba(255,255,255,0.85), rgba(255,255,255,0.85))` overlay on background-image to maintain readability over any background image.

# 2026-04-27

## Booking page timezone awareness

Discovery: Slots already come as UTC ISO strings from the backend, and the frontend's `toLocaleTimeString()` already converts them to the visitor's local timezone. So the visitor already sees correct local times. What's missing is awareness messaging — when host and visitor are in different timezones, there's no indication of this.

Implementation plan:
- Add `timezone` field to User model (IANA format, default "Europe/Berlin")
- API returns `host.timezone` in the booking page response
- Frontend detects visitor timezone via `Intl.DateTimeFormat().resolvedOptions().timeZone`
- If different from host: show info bar with both timezone names + human offset
- Slots continue to display in visitor timezone (already works)

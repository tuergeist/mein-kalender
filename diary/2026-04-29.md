# 2026-04-29

## Ghost events bug — full sync orphan cleanup

Root cause found: Full sync (both Google and Outlook providers) returns all current events in `delta.created`, but `delta.deleted` is always empty. When an event is deleted from the source calendar, it simply stops appearing in the full sync response. But the sync logic only processes explicit deletions from `delta.deleted` — it never compares the full sync set against what's in the DB.

Delta sync handles this correctly via `@removed`/`isCancelled` markers, but if the delta token expires or a full resync is triggered, the orphan events survive indefinitely.

Fix: after processing a full sync (`!syncToken`), compare the set of `sourceEventId`s from the provider response against DB events for that calendar entry. Any DB events not in the provider set are orphans — delete them from the DB and propagate deletions to target calendars.

This also fixes the "QUR Blocker" ghost event reported on the VSX calendar for mail@ch-becker.de. A manual full resync will now clean it up.

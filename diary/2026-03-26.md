# 2026-03-26

## Session: Multi-way sync implementation + landing page review

### Key decisions and discoveries

**Multi-way sync architecture:** Refactored `cloneToTarget()` into `cloneToTarget()` (finds all targets) + `cloneToSingleTarget()` (processes one). Each target runs its full cycle independently — orphan cleanup, create, update. The existing `TargetEventMapping` schema already supported multi-target by design (unique on `sourceEventId + targetCalendarEntryId`).

**Loop prevention — simplest approach wins:** Added `title: { not: { startsWith: "[Sync]" } }` to the Prisma query for unmapped events. No need for origin tracking fields or complex graph detection — the `[Sync]` prefix convention is already enforced on all synced events, so filtering by it is sufficient and debuggable.

**Blocked sync mode:** `buildEventPayload()` helper centralizes the full vs blocked logic for both create and update paths. When blocked, title is `[Sync] Busy`, description and location are null. Source event data stays unchanged — only the clone is redacted.

**Sync targets API vs old target-calendar API:** Kept the old `/api/target-calendar` endpoints untouched for backward compatibility. New `/api/sync-targets` CRUD operates on the same `CalendarEntry.isTarget` flag but supports multiple entries. The old API still reads/writes the first target.

**Outlook recurring events missing titles:** Graph API's `calendarView/delta` endpoint doesn't return `subject` for recurring event instances unless you explicitly use `$select`. Added full field list to the delta query. Also added meaningful fallback titles for events without subjects based on `showAs` (Out of Office, Working Elsewhere, Free, Tentative).

**Calendar source display names:** Default names were raw provider strings ("google", "outlook"). Changed to prettified names ("Google Calendar", "Microsoft 365") with pencil-on-hover hint for inline rename.

**Booking settings page consolidation:** Was 3 separate cards with 3 save buttons. Merged branding, default booking calendar, and working hours into one card with one save. Username keeps its own save (instant, different API). Added note about per-event-type overrides.

**Background overlay opacity:** Was hardcoded at 0.85. Added `backgroundOpacity` field (Float) to both User and EventType. Slider in settings (0-100%) with live preview. Per-event-type override supported.

**Todoist skill improvement:** Tasks were organized by sections (Bugs/Features) not labels. Updated skill to search both labels AND sections, dedup by ID. Found tasks that label-only search missed.

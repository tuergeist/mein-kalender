### Key decisions and discoveries

**K8s env var collision (2026-03-20):** `API_HOST` and `API_PORT` were overridden by Kubernetes service discovery env vars (service named `api` → K8s injects `API_HOST`, `API_PORT`, etc.). Server bound to 127.0.0.1 instead of 0.0.0.0. Renamed to `SERVER_HOST`/`SERVER_PORT`. Important: configmaps also need updating.

**Prisma migrate deploy is live:** Baseline migration `0_init` created from full schema. Dockerfile now uses `prisma migrate deploy`. First deploy required `prisma migrate resolve --applied 0_init` against prod DB. Second migration `1_per_event_type_config` adds `eventTypeId` to `AvailabilityRule` and `bookingCalendarEntryId` to `EventType`.

**Outlook HTML descriptions:** Outlook returns event body as HTML with Exchange boilerplate (`<style>`, `<meta>`, etc.). Added `stripHtml()` on ingest to store clean plain text. Existing events cleaned up on next sync.

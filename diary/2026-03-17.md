# 2026-03-17

## Nav rework, source fetch window, and misc improvements

### Navigation restructure
- Sidebar now has permanent nav links: Calendar, Bookings, Cal Sync, Settings — with active state highlighting
- Burger menu hidden on desktop (sidebar always visible), still works on mobile
- Header cleaned up: removed Bookings/Settings text links, kept logo + admin + avatar dropdown
- Calendar page passes CalendarSidebar as `sidebarContent` below nav links
- Admin pages use `sidebarContent` (renamed from `sidebar` prop)

### New `/sync` page
- Extracted entire target calendar configuration from settings into dedicated `/sync` page
- Includes: target selector, sync period, all filter toggles, source calendar selection, cleanup
- Settings page slimmed to: Calendar Sources (add/rename/disconnect), Booking link, Map Provider

### Source fetch window (`fetchDaysInAdvance`)
- Key insight: no point fetching 365 days of Outlook data if sync/booking only look 30-90 days out
- Added `fetchDaysInAdvance` column on `CalendarSource` (default 90, options: 30/60/90)
- Passed through sync chain: `processSyncJob` → `syncCalendarEntry` → `provider.getEvents`
- Google: now sets `timeMax` on initial sync (previously unlimited forward)
- Outlook: uses configurable value instead of hardcoded 365 days
- Sync page warns when sync period > fetch window and offers to raise it via confirm dialog

### Other changes
- Calendar sources now renamable (click name → inline edit, Enter/blur saves)
- Default sync interval changed from 5min to 10min for new sources
- New OAuth sources trigger immediate initial sync via BullMQ queue
- Per-context calendar selection: many-to-many for target sync + booking availability
- Bookings page has a Configure button linking to booking settings

### Booking page enhancements (from overnight session)
- Booking calendar selector (separate from target sync calendar) with `bookingCalendarEntryId` on User
- Provider email invitations: guest added as attendee → Google/Outlook send invite automatically
- Post-booking redirect: per-event-type `redirectUrl`/`redirectTitle`/`redirectDelaySecs` with countdown timer
- Event type edit modal (was create-only)
- Target calendar settings changed from auto-save to manual Save button

## Booking cancellation detection

User reports: two bookings were cancelled via the calendar (guest declined/deleted the event), but the app still shows them as "confirmed" under "Nächste Buchungen". The calendar itself is correct (events gone), but the Booking model status isn't updated.

This is a sync-detection gap: when a booking's calendar event gets deleted or declined by the guest, the sync job doesn't check whether the event still exists. Need to reconcile booking status with provider event state during sync.

# 2026-03-27

## Session: Todoist backlog clearout + landing page research

### Key decisions and discoveries

**Booking page available days:** Added `GET /api/public/book/:username/:slug/available-days?month=YYYY-MM` that computes slots for every day in a month. Booking calendar now grays out days with no availability instead of showing all future dates as selectable. Both `/book/` and `/B/` pages fetch this on mount and month change.

**Card height stability:** Set `md:min-h-[420px]` on the booking page right panel to prevent the card from resizing when navigating between steps.

**Event type branding override upload bug:** The upload endpoint `/api/profile/image/:type` was updating `User.avatarUrl`/`User.backgroundUrl` even when called from the event type edit page. This made the override invisible because both user and event type ended up with the same URL. Fixed by adding `?skipUserUpdate=true` query param, used by the event type edit page.

**Todoist skill section search:** Tasks were organized by Todoist sections (Bugs/Features) not labels. Updated the skill to search both labels AND sections with deduplication. Found tasks that label-only search missed.

### Changes made
- `public-booking.ts`: available-days endpoint for month-level availability
- Both booking pages: gray out unavailable days, load available days per month
- Both booking pages: min-height on right panel for stable card size
- `user-images.ts`: skipUserUpdate param for event type uploads
- Event type edit page: use skipUserUpdate when uploading branding images
- Todoist skill: search by sections + labels

## Session: Design System Overhaul

### Key decisions and discoveries

**Design consultation:** Researched competitors visually (Cal.com, Calendly, SavvyCal, OneCal, Morgen). Most look interchangeable (white/blue/purple). SavvyCal and Morgen stood out with warmth and personality.

**Deep Rose (#9F1239) as primary:** Initially proposed teal, but that's already used for heykurt.de — need distinct brand identity. Rose is bold and nobody in the scheduling space uses it.

**Warm Amber (#D97706) as accent:** Pairs with rose for a distinctly warm palette. No competitor uses warm accents prominently.

**Typography:** Satoshi (display), DM Sans (body), Geist Mono (data). Moving away from Inter to avoid the "template SaaS" look. Satoshi loaded as local font from fontshare.com woff2 files.

**Subtle grain texture:** SVG noise overlay at 0.03 opacity on body for tactile depth — the "craft" detail that separates from flat SaaS.

**Business positioning:** Lower price point, fast feature delivery, referral-based growth. Reinforces warm/approachable aesthetic over corporate/enterprise.

### Changes made
- Created DESIGN.md (full design system spec) and CLAUDE.md (project instructions)
- `globals.css`: full CSS variable rewrite — rose/amber/stone palette, grain texture, FullCalendar overrides using design tokens
- `tailwind.config.ts`: rose/amber/stone colors, Satoshi/DM Sans/Geist Mono font families
- `layout.tsx`: swapped Inter for Satoshi (local) + DM Sans + Geist Mono
- Added `src/fonts/Satoshi-{Regular,Medium,Bold}.woff2`
- Updated all component color refs: page.tsx, CalendarView, AdminSidebar, LocationMap, oauth-callback

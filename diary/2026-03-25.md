## Session: Todoist backlog clearout + settings reorganization

### Key decisions and discoveries

**Availability rules bug root cause:** `GET /api/availability` was missing `eventTypeId: null` filter, so it returned both default (user-level) and per-event-type rules mixed together. When the booking settings page displayed and re-saved these, it created duplicate default entries. Simple one-line fix.

**Image upload storage decision:** Went with Postgres `BYTEA` (binary in DB) over S3/CDN per the Todoist task description ("store in the db as binary for now"). Used `@fastify/multipart` for file handling. Images served via `/api/public/image/:id` with cache headers. Upload auto-updates `avatarUrl`/`backgroundUrl` on User so existing branding logic just works.

**Settings reorganization approach:** Added sub-navigation to the sidebar (only visible when `section="settings"`) rather than creating a separate layout. Four sub-sections: Calendar Sources (was the main settings page), Booking, Profile (moved from `/profile`), Preferences (map provider). Old `/profile` redirects. All booking sub-pages now highlight Settings in main nav instead of Bookings â€” this was an existing inconsistency.

**Per-event-type branding merge strategy:** Public booking endpoints now do `eventType.brandColor || user.brandColor || null` â€” event-type branding wins, falls back to user defaults. The branding fields are stripped from the `eventType` object in the response to avoid leaking them; only the merged `branding` object is returned. Edit page has a "Branding Override" section with "Use default" placeholders.

**Username change warning:** Only shows when user already has a saved username AND has event types. First-time username setup doesn't trigger the warning since there are no existing URLs to break. Short links still work after username change (they resolve by hash, not username).

**Todoist skills created:** `/list-todoist` and `/solve-todoist` adapted from HeyKurt project. Project ID `6gFMx58H5p8Q9Xcv`. Solve skill does end-to-end: fetch task â†’ implement â†’ test â†’ commit â†’ push â†’ comment on task â†’ mark done.

**Settings > Events separation:** Moved Event Types list out of Booking settings into its own Settings > Events sub-section. Booking settings now only has URL, branding, default calendar, and working hours. Kept `eventTypes` state in booking page for the username change warning check.

**Multi-way sync planning:** Spec'd out the full multi-way sync feature via openspec (proposal + design + specs + tasks). Key design decisions: reuse existing schema (just add `syncMode` field), loop prevention via `[Sync]` title prefix, blocked mode replaces title with "Busy" for privacy. All 4 artifacts complete, ready for `/opsx:apply`.
